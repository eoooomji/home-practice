# 1강 TIL

### 주요 내용

1. ctrl + (`)빽 키 : 뉴 터미널 생성
2. README.md
   > GitHub에 올린 레퍼지터리에서 먼저 보여주는 문서  
   > 이걸 봄으로써 어떠한 형식인지 미리 알 수 있음.
3. md
   > md는 markdown의 줄임말(확장자의 한 종류)
4. .gitignore
   > GitHub에 올리지 않아도 될 환경설정 같은 파일을 만들 때 사용  
   > 레퍼지터리에서 제외 할 파일을 입력하면 올라가지 않음
   - 파일.\*(별표) : 같은 파일명의 모든 확장자를 포함할 때
   - \*(별표).확장자 : 같은 확장자의 모든 파일을 포함할 때
   - (폴더명)/ : 폴더를 제외시킬 때
5. git restore --staged (file)
   > add한 파일 중 stage에서 제외시킬 때
6. git commit --amend
   > 바로 직전 commit한 상태의 파일을 stage로 돌려 편집이 가능하게 만들어 주는 기능.  
   > 추가로 파일을 더 넣어줄 수 있다.(add한 상태의 파일이 stage에 있다면)
7. GitHub 파일 주고 받기(협업)
   > 생성된 레퍼지터리 > settings > collaborators > add people 이름 입력 > GitHub 연동된 메일 동의 > 코드 복사 > 사용

## 프로그래밍

#### 프로그래밍 패러다임

> 프로그램을 작성하는 방식이나 스타일

1. 명령형 프로그래밍(HOW)

- 알고리즘을 명시하고 목표는 명시하지 않는다.
- 프로그래밍의 상태와 상태 변경을 시키는 '구문'의 관점에서 연산을 설명하는 방식이다.
- 절차적 프로그래밍, 객체지향 프로그래밍

2. 선언형 프로그래밍(WHAT)

- 알고리즘을 명시하지 않고 목표만 명시한다.
- 함수형 프로그래밍

#### 프로그래밍 언어

> 컴퓨터 프로그램을 작성하기 위한 언어

1. 절차적 프로그래밍
   - 순차적인 처리 방식으로 프로그램 전체가 유기적으로 연결되도록 하는 프로그램이다.
   - 처리되는 속도가 빠르나 유지보수가 어렵고 실행순서가 정해져 있으므로 코드의 순서가 바뀌면 동일한 결과를 보장하기 어렵다.
2. 객체지향 프로그래밍
   - 실제 세계를 모델링하여 프로그램을 객체들의 모임으로 표현하는 프로그램이다.
   - 클래스를 이용해 연관있는 처리부분과(함수) 데이터부분(변수)을 하나의 객체로 묶어 객체를 생성해 문제를 해결한다.
   - 캡슐화, 상속, 다양성
3. 함수형 프로그래밍
   - 순수함수를 보조함수와 조합하고 소프트웨어를 만드는 방식이다.
   - 로직내에 복합성을 해결하고 변수사용을 억제하여 상태변경을 피하려는 프로그래밍이다.
   - 일급객체, 고차함수, 불변성, 순수함수, 합성함수

> **인터프리터 언어**  
> : 원시코드를 기계어로 변환하는 과정 없이 한 줄씩 바로 명령어를 실행하는 언어  
> **컴파일 언어**  
> : 소스 코드 여러개를 하나로 묶어서 컴파일한 후에 실행하는 프로그래밍 언어

## 문자코드와 인코딩

### 1-1. 문자코드

> 문자와 숫자를 1:1로 매핑시켜놓은 값이다.
> 컴퓨터가 사용하는 0과 1로 바꿀때 사용되는 기준의 코드

### 1-2. 문자코드의 종류

> ASCII코드(7비트), 확장ASCII코드(=ANSI)(8비트), 유니코드, EUC-KR코드

### 2-1. 인코딩

> 문자코드로 바뀐 숫자(0과1)을 컴퓨터에 저장하기 위해 하는 과정
> 쓰인 문자코드에 따라 인코딩 또한 달라진다.

### 2-2. 인코딩의 종류

> ASCII코드(ASCII인코딩), 유니코드(utf-8(가변. 언어에 맞게끔 변함), utf-16(고정. 무조건2바이트만 사용), utf-32)

- code paging : 각 나라별로 제공(cp949) (cmd 들어가서 chcp 입력)

## 진법(진수)

### 1. 진법

> 컴퓨터는 0과 1만을 사용하기에 모든 문자와 숫자가 0과 1로 이루어져야 함
> bit : 0과 1만 표현
> byte = 8bit

### 2. 진법의 종류

> 2진법 : 0과 1로 표현  
> 8진법 : 0 ~ 7로 표현  
> 10진법 : 0 ~ 9로 표현  
> 16진법 : 0 ~ 9 A(10) B(11) C(12) D(13) E(14) F(15)로 표현

### 2. 웹 접근성

> 모든 사람(장애인/고령자/비장애인 등)이 사이트에 쉽게 접근할 수 있도록 구현하는 것

### 4. 웹 표준 기술

> HTML5(웹 페이지 구성) > CSS3(스타일 적용) > JavaScript(=ECMAScript)(사용자 반응 처리 등)

### 5. HTMl 특징

1. 태그는 대소문자 구분이 없다.(but, CSS에서의 속성명은 대소문자 구분 있음)
2. 확장자는 반드시 html, htm로 만들어야 한다.
3. 엔터, 스페이스바, 탭은 작용하지 않는다.
4. 태그의 순서를 반드시 지켜야 웹브라우저가 인식한다.
5. 파일이나 폴더명은 반드시 여백이 없는 영문자여야 한다.
6. HTML의 요소는 블록 요소와 인라인 요소로 나뉜다.
7. 모든 문서의 태그 적용은 위에서 아래로 적용이 된다.(값을 줄 떄 순서가 있고 생략이 가능한 것이 있다.)

## HTML문서 작성 구조

> !DOCTYPE html : HTML5라는 문서 표시

> meta charset="UTR-8" : 인코딩을 UTF-8로 한다는 표시

> title : 홈페이지 상단 탭의 이름

> head : 헤드의 영역은 현재 홈페이지의 실행 환경 및 설정에 대하여 작성하는 부분

> 주석처리 ctrl + / : 해당 부분을 주석처리한다. 소스파일에 그대로 노출됨. 중요한 코드는 주석처리를 하지말고 다른 코드를 사용하여 가린다.

> body : 웹 페이지의 실질적인 내용을 적는 공간.

> style 태그 : 지정한 영역에 스타일을 부여한다. CSS언어로 작성하며 내부스타일과 외부스타일로 나뉜다. (인라인 스타일은 선호하지 않는다.)

> h1 ~ h6 태그 : heading의 줄임말로 문서의 제목을 표시할때 사용되는 태그이다.

> p 태그

- paragraph의 줄임말로 텍스트를 문단으로 정의할때 사용한다.
- 단락 요소에는 a, img등의 인라인 요소와 텍스트만을 사용할 수 있다.
- 단락 요소에는 블록요소를 사용할 수 없다.
- 단락안에는 강제로 줄바꿈을 해야하는 경우 "br/"태그를 사용할 수 있지만, 시각적인 효과등을 위해 남용하지 않는다.

> br/ 태그

- 엔터와 동일하게 줄을 바꿔주는 태그이다.
- "/" 를 붙여줘야 한다.(웹 표준 방식)

> hr/ 태그

- 수평 줄을 삽입하는 태그이다.
- "/" 를 붙여줘야 한다.(웹 표준 방식)

> span 태그 : 인라인의 대표적인 요소이다.

> a 태그

- href에 연결할 주소를 입력하면 된다.
- 하이퍼링크를 연결하는 태그이며, 인라인 요소이다.
- 인라인 요소는 요소간의 줄바꿈이 일어나지 않는다.
- width와 height을 지정해도 실제 표현되지 않는다.
- target 속성을 이용해서 문서가 실행되는 위치를 지정할 수 있다.
  \_parent(상위윈도우) / \_blank(새문서) / \_top(상위프레임) / \_self(현재문서)
- 특정 위치에 연결을 하려거든 id속성을 활용하면 된다.

> div 태그

- 블록의 대표적인 요소이다.
- 블록요소는 요소 앞뒤로 줄바꿈이 일어난다.
- width와 height를 지정할 수 있다.

> q 태그

- 인라인요소와 텍스트를 포함할 수 있다.
- 큰따옴표로 표시가 된다.

> blockquote 태그

- 블록요소와 텍스트만 포함할 수 있다.
- 들여쓰기로 표현이 된다.

> cite 속성

- 인용문을 가져온 주소로 어느 주소에서 가져왔는지 URL을 적어주면 된다.
- 출처를 표시하는것이 예의이다.

> abbr 태그

- 축약어이며 단어의 자음이나 모음만 맞추는 등 긴 단어를 줄여 사용할때 쓰인다.

> pre 태그

- Pre-Fomatted Text 약어로 정형화된 텍스트를 나타내는 블록요소이다.
- 실행을 했을때 입력된 모양 그대로 출력이 된다.

> del 태그

- 삭제된 텍스트를 의미하는 요소이다.(가운데에 줄이 그어짐)

> ins 태그

- 추가된 텍스트를 의미하는 요소이다.(밑줄이 그어짐)

> dl 태그

- 정의형 목록을 표현할때 사용한다.
- 주종관계에 있는 내용을 표현할때 사용한다.
- dt : 타이틀 느낌 / dd : 내용 작성 부분

> img 태그

- src : 그림파일명(이미지 파일이 있는 경로)
- alt : 설명문구(이미지에 대한 설명을 넣음)
- title : 풍선도움말로 나오는 텍스트

> strong 태그

- 강한 강조의 의미를 나타낸다.
- 'b'태그는 '굵게' 나타내는 표현을 위한 요소이다.

> em태그

- 강한 기울기를 의미한다.
- 기울기를 표현할때는 'i'태그를 이용해서 표현한다.
<!-- 위 두 태그 모두 다 강하다는 의미를 지녔다. 시각장애인분들이 이용하는 홈페이지를 읽어주는 스크린리더라는 프로그램 사용시 strong과 em부분은 강조를 해서 읽어준다. -->

> sup 태그

- SuperScript(위첨자)를 의미하는 요소이다.

> sub 태그

- SubScript(아래첨자)를 의미하는 요소이다.

> address 태그

- 웹사이트가 운영되고 있는 회사정보를 표현할때만 address요소를 사용한다.  
  단, 일반 주소를 표현할때는 address요소를 사용하지 않는다.

> Character Entity

- 마크업 했을 때 문제를 발생시킬수 있는 문자들을 표시하기 위한 참조문자를 의미한다.
- https://www.w3school.com/html/html_entities.asp
  https://developer.mozilla.org/en-us/docs/Glossary/Entity (위 사이트에서 확인 가능)

> table 태그

- 표를 만들때 사용
- caption : 표의 제목 생성
- thead : 표의 제일 윗 부분 제목 생성 및 리스트 단위. 표머리
- tfoot : 표의 제일 아랫 부분. 표꼬리
- tbody : 표의 내용이 들어가는 부분. 표본문
- tr : 표의 행
- th : 표의 머릿 부분
- td : 표의 기본 내용 부분
- colspan : 가로 행 병합
- rowspan : 세로 행 병합

> ul, ol 태그

- ul : 비순서적 목록을 표현할때 사용한다.
- ol : 순서가 있는 목록을 표현할때 사용한다.
- 안의 리스트 작성시 'li'태그 사용

> form 태그

- 클라이언트로부터 입력받은 정보를 서버로 보내기 위한 요소를 작성한다.
  - action 속성 : 서버로 요청할 URL(URI)
  - method 속성
    1. get : 폼의 데이터를 서버로 전송할때 256바이트 미만이거나 보안이 필요하지 않는 데이터를 보낼때
    2. post : 폼의 데이터를 서버로 전송할때 256바이트 초과이거나 보안이 필요한 데이터를 보낼때
  - enctype 속성
    1. 전송할 데이터에 첨부파일이 없을때 : application/x-www.form-urlencoded를 지정한다.
    2. 전송할 데이터에 첨부파일이 있을때 : multipart/form-data를 지정한다.
- fieldset : 여러개의 폼 요소를 그룹화 할때 사용한다.
- legend : 폼 요소 그룹화 설정된 것에 대한 제목을 지정한다.
- label : 텍스트와 폼요소의 상관관계를 설정한다.
  - for 속성 : 속성 값으로 포커스를 이동하기 위해서 설정한다.  
    라디오버튼 및 체크박스를 누를때 크기가 작아 누르기 힘든 부분을 글자로 된 부분을 클릭해도 체크가 되기 쉽게 커서를 나오게 하는 기능
    for의 이름과 id의 이름이 동일해야 한다.
- input : 폼에서 입력하는 박스를 만드는 태그.(작성 후 끝에 '/'를 붙여야 한다.)
  - type 속성 : 다양한 속성 값을 선택하는 속성 값
    1. hidden : 사용자에게 보이지 않지만 서버로 넘겨지는 값을 가지는 필드
    2. text : 이름이나 주소 등 입력
    3. password : 비밀번호 입력
    4. radio : 라디오버튼(원형). 라디오 버튼 생성시 다중선택이 되지 않도록 name속성을 같은 값으로 지정해야 한다.
    5. checkbox : 체크박스(다중선택 가능)
    6. submit : 입력한 데이터를 전송하는 기능(value값 필수)
    7. reset : 입력한 데이터를 지우는 기능(value값 필수)
    8. file : 파일첨부
  - placeholder 속성 : 텍스트 안에 글 넣는 기능
  - maxlength 속성 : 텍스트 안에 최대 글자 수 제한 기능

> select 태그

- 셀렉트박스(드롭다운)를 표시한다.
- option : 셀렉트 박스에 포함된 항목들을 정의하기 위해 사용한다.
  - selected 속성 : 여러개의 항목 중에서 선택된 항목을 표시한다.
  - value 속성 : 각 항목의 값을 지정하기 위해 사용한다. 서버로 데이터를 보낼때 value속성의 값이 넘어간다.
  - selected 속성 : 기본적으로 선택된 값을 지정한다.
- optgroup : 셀렉트 박스 안에 그룹을 나누어 목록을 만들때 사용한다.
  - label 속성 : 나뉘는 항목의 항목명을 정한다.

> textarea 태그

- 여러줄을 입력할 수있는 박스를 표시한다.
  - cols & rows 속성 : 텍스트 박스의 크기를 지정하기 위해 사용한다.
  - disabled 속성 : 화면에 표시는 하되 데이터를 수정할 수 없도록 비활성화 상태로 표시한다.

## HTML 문서 작성 구조 -2

> button 태그 : 버튼 속성 값을 가지는 태그

> text-align 속성 : 글씨 정렬 (lefr, middle.right)

> margin 속성 : 텍스트 기준 바깥 여백

> padding 속성 : 텍스트 기준 안쪽 여백

> 레이아웃

- 화면을 분할하거나 배열하여 구성하는 것
- HTML5 웹 표준에서는 각 영역을 구분하는 구조적 태그 요소를 정의하여 사용
  1.  header 태그 : HTML5문서의 머리말 영역으로 중요한 정보르 표시 (예 : 사이트의 제목, 로고 등)
  2.  nav 태그 : 내비게이션 영역으로 웹사이트 내에 분류된 다른 영역으로 이동할 때 사용
  3.  section : 문서의 영역을 구성할 때 사용. header, article 태그 등을 포함할 수 있음
  4.  article : 독립된 주요 콘텐츠 영역을 정의. 하나의 section 태그 내에 여러개의 article 태그를 구성할 수 있음. (섹션과 아티클은 비슷한 개념이므로 서로 대체적으로 사용하기도 한다.)
  5.  aside : 주요 콘텐츠 이외에 남은 콘텐츠를 표시 (예 : 사이드바(sidebar) 등)
  6.  footer : 사이트의 자세한 정보를 표시 (예 : 저작권 정보, 관리자 정보, 회사 정보 등)

> datalist 태그

- html5요소에 새로 추가된 속성으로 form요소에서 미리 지정된 옵션 목록을 제공함
- 사용자들이 특정 글자를 입력하면 그에 해당하는 미리 지정된 목록을 보여줌
- 주의할 점은 input요소에서 list의 속성값과 datalist의 id값이 일치해야 한다.

> type="date"

- 년-월-일 단위로 원하는 날짜를 입력할 수 있음

> type="month" / type="week"

- 년-월 단위로 날짜 입력 / 년-주 단위로 날짜 입력

> type="time" / type="datetime" / type="datetime-local"

- 시간만 입력 / 년-월-일-시간(T붙여야함) 단위로 입력. 직접 입력해야 함 / 자동으로 입력

> type="color"

- 색상 칩에서 원하는 색상을 선택할 수 있음

> type="number"

- 숫자 입력 (최대, 최소 지정 가능)

> type="range"

- 특정 범위의 숫자를 선택할때 사용
- value를 통해 초기 범위 지정 가능
- range바 옆에 숫자 표기를 원하면 자바스크립트를 이용해 만들 수 있다.

> type="email"

- email형식에 맞춰 입력하지 않으면 전송 시 풍선말로 메세지를 띄운다.

> meter 태그

- 범위 안의 값을 지정하는 바를 만들 때 사용
- 숫자 표기를 원하면 자바스크립트를 이용해 만들 수 있다.
- min과 max는 숫자표기 / low와 high는 퍼센트로 표기

> autofocus 속성

- 페이지 출력 시 지정한 곳에 포커스를 맞추는 기능. 속성명과 값이 같다.

> readonly 속성

- 읽기 전용으로 따로 타이핑이 안되고 읽는 것만 가능하게끔 하는 기능. 속성명과 값이 같다.

> disabled 속성

- 비활성 기능으로 만드는 속성. 리액트에서는 읽지를 못한다. 속성명과 값이 같다.

> required 속성

- 자동으로 글귀를 완성 시켜주는 기능. 속성명과 값이 같다.

> autocomplete 속성

- 자동글귀를 on, off 해주는 기능.

> video 태그

- src를 사용하여 파일을 가져온다.
- controls 속성 : 동영상 컨트롤러를 표시해준다.
- autoplay 속성 : 페이지를 키면 자동으로 재생시킨다.
  - 현재 크롬 웹페이지에서는 기능이 실행이 안된다. 사용을 원하면 mute 속성을 추가한뒤 스크립트를 사용해 mute를 해제하면 된다. (chrome viedo autoplay)
- loop 속성 : 반복재생
- 확장자로 인해 비디오 재생이 안되면 다른 확장자의 비디오 파일을 여러개 올린다. 그러면 그 중 재생이 가능한 확장자로 웹페이지에 표시가 된다.
  - source 태그 사용후 sec속성으로 파일을 가져온 다음 type으로 확장자명을 입력해준다.
